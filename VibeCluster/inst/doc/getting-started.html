What is VibeCluster?

VibeCluster answers a simple question: “Given the way a song sounds, what mood does it create — and what other songs will make me feel the same way?”

It uses three statistical methods from course:

Method Role
K-Means Clustering Groups songs into k mood clusters
PCA Projects 6-D features to 2-D for the Mood Map
kNN Finds the n songs closest to your mood profile

Step 1 — Load the Dataset

library(VibeCluster)

songs <- load_songs()
print(songs)
#> === VibeCluster Song Dataset ===

The built-in dataset has 364 songs across 6 mood archetypes with 6 audio features each (energy, valence, danceability, tempo, acousticness, loudness).

Step 4 — The Mood Map (PCA)

plot_mood_map(result)

Each dot is a song; diamonds mark cluster centroids; convex hulls outline each mood’s territory. PC1 captures roughly the energy–acousticness axis and PC2 captures valence.

Step 6 — Personalised Recommendations (kNN)

Define your mood profile as a 6-element feature vector, then ask for the closest songs:

# High-energy, dark mood — Angry/Energetic profile
my_profile <- c(energy        = 0.90,
                valence       = 0.15,
                danceability  = 0.55,
                tempo_norm    = 0.85,
                acousticness  = 0.05,
                loudness_norm = 0.92)

recs <- recommend_songs(result, profile = my_profile, n = 8)

Bringing Your Own Data

Export tracks from the Spotify Web API (or use the spotifyr package), normalise the features, and pass the CSV to load_songs():

my_songs <- load_songs("my_spotify_tracks.csv")
result   <- mood_cluster(my_songs, k = 6)

Your CSV only needs the six feature columns; title and artist are optional but will appear in recommendation output.

Function Reference

Function Description
load_songs() Load built-in or custom CSV dataset
mood_cluster() K-means clustering into mood groups
plot_mood_map() PCA Mood Map scatter plot
get_mood_palette() Bar chart + feature heatmap
recommend_songs() kNN personalised recommendations
mood_summary() Elbow + silhouette plot for choosing k
